Лабораторная работа №4  
Веб-приложение с регистрацией пользователей (FastAPI + React + PostgreSQL)

Описание проекта

Данный проект представляет собой
веб-приложение с регистрацией пользователей.

Архитектура проекта разделена на три основные части:
- Backend — FastAPI (Python)
- Frontend — React (Vite)
- База данных — PostgreSQL

База данных

Используется PostgreSQL.

При первом запуске контейнера автоматически:
- создаётся база данных "paszi"
- создаётся таблица "users"

Структура таблицы "users":
- "id" — идентификатор пользователя
- "login" — уникальный логин
- "password_hash" — хэш пароля (Argon2)
- "created_at" — дата регистрации

Backend (FastAPI)

Backend реализует REST API для регистрации пользователей.

Основные возможности:
- регистрация нового пользователя
- валидация логина и пароля
- хэширование паролей с использованием Argon2
- обработка ошибок (409, 422)
- health-check эндпоинт /health

Backend запускается на порту 8000.

Frontend (React)

Frontend реализован с использованием React + Vite.

Функциональность:
- форма регистрации пользователя
- отправка данных на backend
- отображение ошибок и успешных сообщений
- пользовательские сообщения об ошибках на русском языке

Frontend доступен на порту 5173.

Безопасность

- пароли не хранятся в открытом виде
- используется современный алгоритм хэширования Argon2
- все входные данные проходят серверную валидацию (Pydantic)

Тестирование

Для backend реализованы тесты с использованием pytest и FastAPI TestClient.

Проверяются сценарии:
- успешная регистрация пользователя
- попытка регистрации с уже существующим логином
- регистрация с некорректным (слабым) паролем

Запуск проекта

В корне проекта выполнить команду:

bash docker compose up --build
